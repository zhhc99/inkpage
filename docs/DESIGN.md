# inkpage 设计

## 项目定位

inkpage 应该是一个轻量, 简洁的 web 墨迹画板.

## 核心原则

- 支持鼠标, 触控, 手写笔.
- 墨迹流畅自然, 适合书写, 无论设备是否有压感.
- 操作逻辑符合直觉, 用户心智负担低.
- 中英文本地化.
- 适应性布局.

## 交互

inkpage 将操作分为三类:

- 内容操作: 墨迹, 橡皮, 选择
- 视图操作: 平移, 缩放
- 临时动作: 不切换主工具的临时覆盖行为

### 一般输入语义

- 鼠标左键和笔尖始终执行当前主工具.
- 右键和手写笔侧键始终是临时橡皮.
- 中键, 空格拖动, 双指拖动始终是平移.
- 滚轮和双指缩放始终是缩放.
- 统一缩放开启后, 缩放被锁定, 但平移仍然可用.

### 触摸语义

单指触摸默认用于:

- 选择工具下: 选择对象
- 墨迹 / 橡皮下: 移动画布

触摸墨迹是一个显式开关. 开启后, 墨迹 / 橡皮下, 单指触摸将被用于执行工具 (而非移动画布).

### 选择工具

包含如下功能:

- 点击笔画: 单选
- 点击空白: 清空选择
- 从空白拖动: 框选完全包含在区域内的笔画
- 拖动已选内容: 移动当前选择

## 工具栏

工具栏按语义分组:

1. 内容工具: 墨迹, 橡皮, 选择
2. 墨迹设置: 颜色, 墨迹微调
3. 模式开关: 触摸墨迹, 模拟压感, 统一缩放
4. 文档操作: undo, redo, 重置, 项目

## 界面原则

界面遵循 Material Design 3 规范.

- 保持清晰的层级, 阴影和圆角.
- 使用克制的动效和反馈.
- 界面支持横屏和竖屏, 适应性布局.

界面设计服务于书写, 不应喧宾夺主.

## 工程原则

项目采用 `index.html + src/*.ts + esbuild` 的轻量架构.

- DOM 操作保持原生命令式风格
- 不引入框架
- 不引入运行时依赖

## 文档规范

- 所有文档使用中文汉字书写, 标点符号使用英文标点.
- 文档保持简洁, 系统化, 不冗余.

## 代码规范

- 代码应尽可能简洁.
- 尽量不使用注释, 让代码自述功能. 如果需要注释, 使用英文注释.
