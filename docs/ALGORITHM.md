# inkpage 墨迹渲染算法

inkpage 采用五阶段矢量管线, 将原始输入点转换为闭合多边形轮廓, 最终作为填充路径渲染到 Canvas. 无论设备是否具备压感, 均能产出流畅自然的笔画.

## 输入

输入为有序的时间序列采样点, 每个点包含三个分量: `[x, y, pressure]`. 其中 pressure 为 0~1 的浮点数, 由 `getEffectivePressure()` 根据当前压感模式注入: 模拟压感模式下为 null (由阶段三通过速度计算), 原生压感模式下为 PointerEvent 的真实压力值, 无压感设备回退到固定值 (见阶段三).

为了提高采样密度, 优先使用 `getCoalescedEvents()` 获取合并事件列表. 对于手写笔, 移动距离小于 `0.2 / zoom` 的连续点会被合并, 以消除抖动.

## 阶段一: 路径平滑

对原始输入点施加指数移动平均 (EMA) 平滑, 消除手部震颤带来的噪声.

对每个非首点, 将其坐标向前一个已平滑点做线性插值:

```
smoothed = lerp(raw, prev, 1 - t)
```

其中 `t = 0.15 + (1 - smoothness) * 0.85`, smoothness 为可调参数 (默认 0.5). t 值越小, 平滑越强.

**特殊处理**: 完成笔画的最后一个点跳过平滑, 直接使用原始坐标, 确保笔画末端精确落在抬笔位置.

平滑后, 同时记录每个点到前一点的距离 (distance) 和从起点到当前点的累积路径长度 (length).

## 阶段二: 方向与法线计算

对平滑后的点序列, 使用中心差分法计算每个点的方向向量:

```
direction = normalize(next - prev)
```

- 首点的 prev 用自身代替 (退化为 forward difference)
- 末点的 next 用自身代替

将方向向量旋转 90 度得到法线向量:

```
perpendicular = (-direction.y, direction.x)
```

法线方向指向笔画轮廓的左侧.

## 阶段三: 压力到半径映射

将每个点的压力值转换为绘制半径, 需要处理两种输入场景.

### 压力来源

压力来源由用户模式决定, 与输入设备类型无关:

- **模拟压感模式 (默认关闭)**: 无论使用手指、手写笔还是鼠标, 均通过速度模拟压力. `simulatedPressure = 1 - min(1, distance * sensitivity / size)`. 其中 sensitivity 为可调参数, 默认 1.0. sensitivity 越大, 笔画会在更低速度时开始变细, 且变细更快.
- **原生压感模式 (默认开启)**: 优先读取 PointerEvent 的 hardware pressure; 若设备不支持压感 (pressure 为 0), 则回退到常量 0.8, 产生均匀粗细的笔画. 乘以 1.05 放大系数后截断到 [0, 1].

两种模式对所有输入设备使用相同的缓动函数和渲染参数, 确保手指与手写笔的笔画粗细一致.

### 低通滤波

对压力序列施加一阶低通滤波, 消除突变:

```
filtered = lerp(current, prevFiltered, 0.275)
```

0.275 为固定系数, 平衡响应速度和平滑度.

### 半径计算

使用固定的内部参数将压力映射为半径:

```
radius = minRadius * easing(0.5 - thinning * (0.5 - pressure))
```

- `minRadius` = size / 2
- `thinning` 固定为 0.5, 不作为 UI 可调参数暴露
- 缓动函数: `easing(t) = sin(t * PI/2)` (纯正弦, 起笔柔和)

最终半径钳制到 `max(0.5, radius)`, 并确保首末点半径不低于 1.

## 阶段四: 轮廓生成

根据每个点的坐标, 半径和法线方向, 生成闭合的多边形轮廓. 包含两种线帽模式.

### 侧边偏移

对每个点, 沿法线方向向两侧偏移:

```
left[i]  = center + perpendicular * radius
right[i] = center - perpendicular * radius
```

得到左轮廓和右轮廓两条点列.

### 单点处理

当输入仅有一个点时:

- Round 模式: 生成 12 个顶点的正多边形近似圆
- Pointed 模式: 生成 4 个顶点的菱形

### Round 线帽

在笔画首末两端各添加半圆弧. 弧线通过角度参数化生成, 每端 10 段.

- 起笔弧: 从右轮廓起点方向, 逆时针扫过 PI 弧度, 到达左轮廓起点方向
- 收笔弧: 从左轮廓末端方向, 顺时针扫过 PI 弧度, 到达右轮廓末端方向

最终轮廓按以下顺序拼接: 左轮廓(正向) → 收笔弧 → 右轮廓(反向) → 起笔弧. 闭合多边形由 SVG 的 Z 命令封闭.

### Pointed 线帽

沿首末点的方向向量各延伸一个半径长度, 形成钢笔尖风格的锋利端点:

```
tipStart = firstCenter - direction * radius
tipEnd   = lastCenter  + direction * radius
```

左轮廓起点和右轮廓起点合并到 tipStart, 左轮廓末端和右轮廓末端合并到 tipEnd.

最终轮廓: 左轮廓(含 tipStart) → 右轮廓(含 tipEnd, 反向).

## 阶段五: 贝塞尔路径渲染

将轮廓点列转换为 SVG 二次贝塞尔曲线路径字符串, 通过 Path2D API 填充渲染.

### 曲线生成规则

对于轮廓点序列 `P[0], P[1], ..., P[n-1]`:

- 从 `P[0]` 移动画笔开始 (`M` 命令)
- 对 `i = 1` 到 `i = n-3`, 以 `P[i]` 为控制点, 以 `P[i]` 和 `P[i+1]` 的中点为通过点, 生成二次贝塞尔曲线 (`Q` 命令)
- 最后一段以 `P[n-2]` 为控制点, 直接到达 `P[n-1]`
- 闭合路径 (`Z` 命令)

这种"中点技术"保证相邻曲线段在连接点处切线方向一致, 产生视觉上光滑的轮廓.

### 短轮廓回退

当轮廓点数较少时, 路径生成会回退为近似圆形的闭合路径, 避免退化轮廓.

## 渲染管线

### 文档空间与无限画布

笔画坐标存储在无限文档空间中, 不存在硬性边界. 在无已保存视图需要恢复时, 初始视口将坐标原点 (0,0) 置于屏幕中心; 若已有笔画内容, 则自动居中到内容边界. 平移和缩放通过 `state.panX/panY` 和 `state.zoom` 控制, 用户可以无限平移到任意位置绘画.

**PNG 导出** 自动计算所有笔画的轴对齐包围盒的并集, 加 40px 内边距, 作为导出图片的尺寸. 坐标原点不再参与导出逻辑.

### 双画布缓存

已完成笔画渲染到离屏 Canvas 并缓存. 每帧只需重绘:

1. 白色背景 + 网格点阵
2. 离屏缓存 (已提交笔画, 整图拷贝)
3. 当前笔画, 选择高亮和交互叠加层 (实时渲染)

已提交笔画会在收笔时缓存 `bbox` 和对应的单点半径或 `Path2D`. 若视图未变化, 新提交笔画直接追加到离屏 Canvas, 避免整图重建.

缓存失效条件: 笔画擦除, 项目载入, 画布重置, 选择内容移动, 平移变化, 缩放变化, 画布尺寸变化.

### 视口裁剪

每个已提交笔画记录轴对齐包围盒 (bbox). 渲染时先检查 bbox 与当前视口是否相交, 不相交则跳过.

### 帧调度

所有渲染请求通过 `requestAnimationFrame` 合并, 每帧最多执行一次完整渲染. 自动保存延后到空闲时机写入 `localStorage`, 避免阻塞后续书写.
